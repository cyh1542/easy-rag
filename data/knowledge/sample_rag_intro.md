# RAG 简介示例

RAG 是 Retrieval-Augmented Generation 的缩写，中文通常翻译为“检索增强生成”。

一个典型的 RAG 系统通常包含下面几个步骤：

1. 先把本地知识库文档切分成多个片段。
2. 再使用 embedding 模型把这些片段转换成向量。
3. 把向量和对应文本一起存入向量数据库。
4. 用户提问时，先把问题也转换成向量。
5. 在向量数据库里检索和问题最相关的若干片段。
6. 把检索到的内容连同用户问题一起发给大语言模型。
7. 大语言模型基于检索结果生成最终答案。

RAG 的优点是可以让大模型利用外部私有知识，减少幻觉，并且不需要频繁微调模型。
